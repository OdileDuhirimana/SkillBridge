# Deployment Guide (Render + Vercel)

## Targets

- Backend API on Render (repository root)
- Frontend on Vercel (`client/`)

## Render Backend (`skillbridge-api`)

`render.yaml` is included.

Required env vars:
- `MONGODB_URI` (MongoDB Atlas connection string)
- `CLIENT_URL` (Vercel frontend URL)

Security:
- `JWT_SECRET` is auto-generated by Render
- `JWT_EXPIRE` default is `7d`

Optional integrations (set only when enabled):
- OpenAI: `OPENAI_API_KEY`
- Cloudinary: `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`
- Email SMTP: `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USER`, `EMAIL_PASS`
- Firebase push: `FIREBASE_PROJECT_ID`, `FIREBASE_PRIVATE_KEY`, `FIREBASE_CLIENT_EMAIL`

## Vercel Frontend (`client/`)

Deploy the `client` folder as a Vercel project.

Required env var:
- `REACT_APP_API_URL` set to Render API URL root, e.g. `https://skillbridge-api.onrender.com`

`client/vercel.json` enables SPA route rewrites.

## How To Get Values

- `MONGODB_URI`: MongoDB Atlas > Database > Connect > Drivers
- `CLIENT_URL`: Vercel deployment URL
- `OPENAI_API_KEY`: OpenAI dashboard API keys
- `Cloudinary` keys: Cloudinary console > API keys
- `EMAIL_*`: your SMTP provider settings (Gmail app password, SendGrid SMTP, etc.)
- `Firebase` service-account values: Firebase console > Project settings > Service accounts
